Konsep Dasar Proses Pemrograman

Pemrograman bukan hanya tentang menulis kode, tetapi tentang proses sistematis untuk menyelesaikan masalah

? Apa itu Proses Pemrograman?

Proses pemrograman adalah metodologi terstruktur yang digunakan untuk mengembangkan solusi perangkat lunak. Ini melibatkan serangkaian tahapan yang saling terkait untuk memastikan bahwa program yang dihasilkan dapat menyelesaikan masalah dengan efektif dan efisien.

🎯 Tujuan Utama:

  • • Menghasilkan solusi yang tepat dan akurat
  • • Meminimalkan kesalahan dalam pengembangan
  • • Memastikan kualitas dan maintainability kode
  • • Mengoptimalkan waktu dan sumber daya

⚡ Mengapa Penting?

✓

Mengurangi Kompleksitas

Memecah masalah besar menjadi bagian-bagian yang lebih mudah dikelola

✓

Meningkatkan Kualitas

Setiap tahap memiliki validasi untuk memastikan kualitas output

✓

Memudahkan Kolaborasi

Tim dapat bekerja sama dengan pemahaman yang sama

🧪 Menguji Program

Testing - Validasi dan Verifikasi

🧪

🎯 Tujuan Testing

Testing bukan hanya tentang menemukan bug, tetapi juga memastikan bahwa program memenuhi semua requirement yang telah didefinisikan di tahap analisis dan berfungsi dengan baik dalam berbagai kondisi.

🔍 Jenis-jenis Testing:

🔧 Unit Testing

Menguji komponen individual secara terpisah

🔗 Integration Testing

Menguji interaksi antar komponen

👤 User Acceptance Testing

Validasi dari perspektif pengguna akhir

⚡ Performance Testing

Menguji kecepatan dan efisiensi program

📊 Strategi Testing

✅ Positive Testing

Menguji dengan input yang valid dan sesuai ekspektasi

❌ Negative Testing

Menguji dengan input yang tidak valid atau ekstrem

🎯 Boundary Testing

Menguji nilai-nilai di batas minimum dan maksimum

🔄 Regression Testing

Memastikan perubahan tidak merusak fungsi yang sudah ada

📋 Test Case Design:

  • • Definisikan input dan expected output
  • • Cover semua path dalam algoritma
  • • Test edge cases dan error conditions
  • • Dokumentasikan hasil testing

Studi Kasus: Program Identifikasi Jenis Segitiga

Mari kita terapkan 4 tahap proses pemrograman untuk membuat program yang mengidentifikasi jenis segitiga

🎯 Deskripsi Masalah

Kita akan membuat program yang dapat mengidentifikasi jenis segitiga berdasarkan panjang ketiga sisinya. Program harus dapat membedakan antara:

🔺

Sama Sisi

Ketiga sisi sama panjang

📐

Sama Kaki

Dua sisi sama panjang

📏

Sembarang

Ketiga sisi berbeda panjang

🔧 Demo Interaktif Program

📐
Masukkan nilai sisi segitiga
Klik tombol analisis untuk melihat hasilnya

1 Tahap Analisis (Analyzing)

📋 Problem Statement

🎯 Tujuan Utama:

"Mengembangkan program yang dapat mengklasifikasikan jenis segitiga berdasarkan panjang ketiga sisinya dengan akurasi 100% dan memberikan feedback yang informatif kepada pengguna."

📊 Success Criteria:
  • • Klasifikasi benar untuk semua input valid
  • • Deteksi error untuk input invalid
  • • Interface yang mudah dipahami
  • • Performa responsif (< 1 detik)
📥 Spesifikasi Input
📋 Data yang Diperlukan:
  • • Sisi A: Panjang sisi pertama segitiga
  • • Sisi B: Panjang sisi kedua segitiga
  • • Sisi C: Panjang sisi ketiga segitiga
🔢 Format & Validasi:
  • • Type: Bilangan real positif
  • • Range: 0.1 ≤ nilai ≤ 1000
  • • Format: Desimal (max 2 digit)
  • • Validasi: Harus memenuhi triangle inequality
⚙️ Spesifikasi Proses
🔍 Langkah Pemrosesan:
  1. Validasi input (positif & numerik)
  2. Periksa triangle inequality theorem
  3. Bandingkan panjang ketiga sisi
  4. Klasifikasi berdasarkan kesamaan sisi
  5. Generate output sesuai hasil
🧮 Algoritma Klasifikasi:
  • • IF a = b = c → Sama Sisi
  • • ELSE IF a = b OR b = c OR a = c → Sama Kaki
  • • ELSE → Sembarang
📤 Spesifikasi Output
✅ Output Valid:
  • • "Segitiga Sama Sisi" - Ketiga sisi sama
  • • "Segitiga Sama Kaki" - Dua sisi sama
  • • "Segitiga Sembarang" - Ketiga sisi berbeda
❌ Output Error:
  • • "Input tidak valid" - Nilai ≤ 0
  • • "Bukan segitiga valid" - Triangle inequality
  • • Format: String dengan kapitalisasi konsisten
⚠️ Spesifikasi Constraints
🔒 Batasan Teknis:
  • • Precision: JavaScript floating point (15-17 digits)
  • • Performance: Response time < 100ms
  • • Memory: Minimal memory usage
🌐 Batasan Lingkungan:
  • • Platform: Modern browser (ES6+)
  • • User: Pemahaman dasar geometri
  • • Range: Nilai praktis untuk pembelajaran

🔬 Hasil Analisis Mendalam

🎯 Complexity Assessment:

Problem ini memiliki kompleksitas rendah dengan logika conditional sederhana, cocok untuk pembelajaran algoritma dasar.

📈 Scalability Consideration:

Algoritma dapat diperluas untuk klasifikasi segitiga berdasarkan sudut atau perhitungan luas/keliling.

🔧 Implementation Strategy:

Pendekatan modular dengan fungsi terpisah untuk validasi, klasifikasi, dan error handling.

2 Tahap Desain (Problem Solving)

📖 Algoritma Narasi

  1. 1. Baca nilai panjang sisi a, b, dan c
  2. 2. Validasi bahwa semua nilai positif
  3. 3. Periksa apakah membentuk segitiga valid
  4. 4. Jika a = b = c, maka "Sama Sisi"
  5. 5. Jika tidak, periksa apakah ada dua sisi yang sama
  6. 6. Jika ya, maka "Sama Kaki"
  7. 7. Jika tidak, maka "Sembarang"
  8. 8. Tampilkan hasil

💻 Pseudocode

ALGORITMA IdentifikasiSegitiga
INPUT: a, b, c (real)
OUTPUT: jenis_segitiga (string)

BEGIN
    READ a, b, c
    
    IF (a <= 0 OR b <= 0 OR c <= 0) THEN
        OUTPUT "Input tidak valid"
        RETURN
    END IF
    
    IF (a+b <= c OR a+c <= b OR b+c <= a) THEN
        OUTPUT "Bukan segitiga valid"
        RETURN
    END IF
    
    IF (a = b AND b = c) THEN
        jenis_segitiga = "Sama Sisi"
    ELSE IF (a = b OR b = c OR a = c) THEN
        jenis_segitiga = "Sama Kaki"
    ELSE
        jenis_segitiga = "Sembarang"
    END IF
    
    OUTPUT jenis_segitiga
END

3 Tahap Implementasi (Coding)

💻 Implementasi JavaScript

function identifikasiSegitiga(a, b, c) {
    // Validasi input
    if (a <= 0 || b <= 0 || c <= 0) {
        return "Error: Nilai sisi harus positif";
    }
    
    // Periksa ketentuan segitiga
    if (a + b <= c || a + c <= b || b + c <= a) {
        return "Error: Bukan segitiga valid";
    }
    
    // Klasifikasi jenis segitiga
    if (a === b && b === c) {
        return "Segitiga Sama Sisi";
    } else if (a === b || b === c || a === c) {
        return "Segitiga Sama Kaki";
    } else {
        return "Segitiga Sembarang";
    }
}

// Contoh penggunaan
console.log(identifikasiSegitiga(3, 3, 3)); 
// Output: "Segitiga Sama Sisi"

console.log(identifikasiSegitiga(2, 2, 3)); 
// Output: "Segitiga Sama Kaki"

console.log(identifikasiSegitiga(3, 4, 5)); 
// Output: "Segitiga Sembarang"

🔧 Penjelasan Kode

📥 Input Validation

Memastikan semua nilai sisi positif dan membentuk segitiga valid

🔍 Triangle Inequality

Memeriksa apakah jumlah dua sisi lebih besar dari sisi ketiga

🎯 Classification Logic

Menggunakan conditional statements untuk mengklasifikasi jenis segitiga

🛡️ Error Handling

Menangani kasus input yang tidak valid dengan pesan error yang informatif

  • • Validasi input di awal untuk early return
  • • Pesan error yang deskriptif untuk debugging
  • • Struktur if-else yang logis dan mudah dibaca
